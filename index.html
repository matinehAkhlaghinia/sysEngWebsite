Project Brief

Our team is developing an application on a digital assistant or chatbot to be able to book gym classes for a user almost instantly. For this project, we are working with Nuffield Health and Microsoft. The project will run over the course of a year and will terminate with a user-tested deliverable. As a team, we want to create something which will make users want to go to the gym

Problem Statement

A user needs to book a gym class quickly. He is lazy and does not want to interact with a mobile or online platform which takes concentration and time. A chatbot or a digital assistant application simplifies that by enabling a fast and easy interaction. The user now does not have to look around for all the elements on a platform and can just ask an artificial robot to find everything for him.

What is considered a success

A successful deliverable for this project is one which works on is considered ready to be deployed by our client. Furthermore, we would also like our client to be content with our work. Finally, in order for this project to be successful each team members should have learned new skills and enriched its understanding of software development.

System functionality

For this project, we are building on top of a current platform which is available online and enables user to book gyms classes. We are using Node.js to create a Wrapper API around the existing backhand to make it fit the requirements of the AI user interaction text. On top of this wrapper, we are going to have our digital assistant or chat bot written in the language of the (framework we will use). The exchange of data is done using a RESTFUL approach.
