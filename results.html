Client Requirement Based Comparison:

From the client product's requirements, we can define a list of user stories the digital assistant and chat bot will have to support. This list will enable us to compare the different assistants and bots available to go towards writing the proposals for the project. These user stories are based for gym user of Nuffield Health.

A. Does it support login and logout within the platform
User Story: As a gym member, I want to login only once and have the platform recognise me when I start to converse with the bot or digital assistant. Only if I logout shall I log back in.
B. Can it prompt a user for a task/push notifications/reminders
User Story: As a gym member, I want to receive notifications when I have a class coming up that I booked in the system. As a gym administrator, I want to send notifications to gym members to remind them to give feedback on the gym classed they have just attended.
C. Does it support Analytics
User Story: As a gym administrator, I want to be able to have a record of the number of interactions with the bot to be able to modify it according to user needs. Also, I want to view how useful the tool is.
D. Does it have a speech recognition tool
User Story: As a gym member, I want to be able to talk to the bot and have it book a class for me only using my voice.
E. Does it have a text recognition tool
User Story:As a gym member, I want to be able to converse with the bot by simply texting it.

Technical Tools Based Comparison:

In this part of the comparison, we compare the different SDKs and frameworks available for each of the tools. Find below a list of the frameworks/SDK and the bots that work with them:

Chatbots

  • Microsoft Bot Framework: FB, Slack and Skype. This SDK is provided by Microsoft and enables a developer to build a chat which works for all 3 platforms without having to re-write code for each of them.
  • Api.ai : This is a tool which let s you build bots on
  • ChatFuel: FB. With this tool, it is possible to build a FB bot without doing any coding.

Digital Assistants

  • Alexa Skill kit: Alexa. This is an Amazon tool and enables the creation of alexa skill
  • Cortana development kit: Cortana. Not available to the public yet, however it is possible to request an early version. The very documentation is limited as a result.

We will compare the frameworks on the following criteria:

  1. Preview and publication. How easy is it to preview and publish using this Framework.
  2. Documentation. Is the framework well documented, are there tutorial videos available online showing how to work with them.
  3. Channels. How many different messaging platforms the framework works with.

Comparison Conclusion:

To conclude, we have decided to use the following for each of the proposals:

  • Proposal 1 (digital assistant solution): We will be developing an Alexa Skill. Cortana Dev Kit is not even public yet which makes it very not attractive to us.
  • Proposal 2 (chatbot solution): We will be doing a FB, Skype and Slack bot app using the Microsoft Bot Framework. Microsoft Bot FrameWork is the best of the frameworks out there and considering that we have help from MS, it is the logical one to use.
