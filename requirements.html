Project Background

Many people are now very busy and lead lives on the go and often are on their mobile devices. For them making bookings can be a hassle and time consuming so we are trying to find a way to make this process more interactive and much faster for the customers.

We had met with Alex Matei from Nuffield and have worked closely with representatives from Microsoft such as Paul Thomas in order to develop our chatbot and digital assitant to try address this problem.

Fuller details on the problem statement and solution are given below.

Problem Statement and brief

Project Brief

Our team is working in conjunction with Alex from Nuffiled and also Microsoft in order to develope a bot which will be deployed over various platforms such as Facebook Messenger and Skype in order to allow the customers of Nuffield Health to book gymn classes and eventually expanding to use other services provided by the company.

The project will then continue as we will try to integrate a digital assistant so rather than type, a user will be able to speak to the bot - to achieve this, we will be using digital assistant services such as Cortana or Siri.

The end product will be a bot which allows for customers to better interact with the company and any bookings they wish to make. It is worth noting this would be an alternative to using the website (where bookings have just gone live) or call the centres in order to book sessions and is one of the key motivations behind why this project is being undertaken.

Problem Statement

Most people are usually busy and booking appointments by taking time out to traverse a website or call in to a reception desk is a time consuming task. Nuffiled Health wants to address this issue with their bot as it would allow the clients of Nuffield Centres to interact much more easily with the company. As a result, the clients will be able to book gym sessions or use other services much more easily. One of the gains would be that staff will not have to keep taking as many calls to deal with booking queries since the clients can book and also check or cancel appoitments from the devices. Our challenge would be to find an effective way to encapsulate the different queries and tasks a user may wish to undertake with this bot into a design that facilitates this easier interaction between Nuffield and the clients.

The bot will enable Nuffield to cater for more ways through which customers can book appointments by expanding the online booking service - which increases reach and accessibilty

Reduce the number of missed bookings by reminding people beforehand and also offering the option to cancel.

Goals for success

A successful deliverable for this project is one where we manage to pull together the needs of the user (being able to book, cancel, modify and view appointments) and integrate this with a bot which is easier and more time-efficient to use than traversing the Nuffield website or calling the reception desks. This then needs to be supplemented with a digital assistant that is flexible enough to parse user speech and works for commands which is fair to be expected from normal usage and is not rigid in the inputs it can take. Putting these together would give us a deliverable which the client had outlined in the brief and would be considered a success.

  • Ensure the requirements obtained from the client are clear
  • Complete all of the MUST and SHOULD have requirements
  • This is a fairly new area so we must be realistic with expectations and deliverables promised
  • Carry out thorough HCI design to inform our final design

System functionality

For this project, we are building on top of a current platform which is available online and enables user to book gyms classes. We are using Node.js to create a Wrapper API around the existing backhand to make it fit the requirements of the AI user interaction text. On top of this wrapper, we are going to have our digital assistant or chat bot written in the language of the (framework we will use). The exchange of data is done using a RESTFUL approach.

Gathering Requirements

For gathering more specific requirements, we interviewed some of the Nuffield Health Center clients, this resulted to the personas and scenarios we made which then made us to go back to our requirements and add some features to it and refine it accordingly. The new requirements included:

  1. Getting users' feedback after attending the gym through the bot
  2. Being able to tell the user where the closest Nuffield gym is through both bot and digital assistant
  3. Sending reminders to the users prior to their gym class through both chat bot and digital assistant

THe interviews were semi- structured, with some questions considered as vital being asked of everybody and other questions were more opinion based and left open-ended. Once the interviews had been conducted the results were collated and analysed to pick out what mattered the most to users and also what we had presumed was important but turned out not to be. The analysis was done formally as the set questions were on a scale of 1-10 and this made it easy to compare between responses while for the open ended questions a list of key requirements were written out for each response.

Requirements

After discussing the project with Nuffield Health Center as our client, in the first stage, we came up with a set of requirements considering the general users of Nuffield Gym. Their problem was that some people found it hard and time consuming to open their laptop and go through the website and do the gym class booking, hence they would stop going to the gym easily. They wanted to solve this issue by using bots and digital assistants and providing the booking functionality as well as sending reminders to them to stop them from quitting the gym.

User cases

Client Requirement Based Comparison:

From the client product's requirements, we can define a list of user stories the digital assistant and chat bot will have to support. This list will enable us to compare the different assistants and bots available to go towards writing the proposals for the project. These user stories are based for gym user of Nuffield Health.

A. Does it support login and logout within the platform
User Story: As a gym member, I want to login only once and have the platform recognise me when I start to converse with the bot or digital assistant. Only if I logout shall I log back in.
B. Can it prompt a user for a task/push notifications/reminders
User Story: As a gym member, I want to receive notifications when I have a class coming up that I booked in the system. As a gym administrator, I want to send notifications to gym members to remind them to give feedback on the gym classed they have just attended.
C. Does it support Analytics
User Story: As a gym administrator, I want to be able to have a record of the number of interactions with the bot to be able to modify it according to user needs. Also, I want to view how useful the tool is.
D. Does it have a speech recognition tool
User Story: As a gym member, I want to be able to talk to the bot and have it book a class for me only using my voice.
E. Does it have a text recognition tool
User Story:As a gym member, I want to be able to converse with the bot by simply texting it.
