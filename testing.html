Experiments for bots

Exp # Experiment Title Experiment details Experiment results Success and failure Done by Date
1 Skype Bot SkyScanner - Typo I will intentionally make typos in my queries for commands (such as new search) to see whether or not Skype Bot is able to deal with simple and reasonable typo errors. Any commands given to the bot have to be written word for word and spelt exactly correctly else the bot is unable to process this, even if only a single letter is off. Failure Sachchit Prasad 02/12/2016
2 Skype Bot SkyScanner - Typo I will intentionally make typos in my queries for cities searched (such as New Yrk for New York and Belfst for Belfast) to see whether or not Skype Bot is able to deal with simple and reasonable typo errors. The bot is able to correct for these typos and will return the intended city searched for. At most you can make a typo of two letters (whether changed or omitted, anything beyond this will not work) - this is reasonable as typos aren't likely to be beyond this level. Success Sachchit Prasad 02/12/2016
3 Skype Bot SkyScanner - Garbage information I shall query something which does not answer the question asked and does not make sense for a user to put in as an answer. For example, on the prompt for "What is the start date", I shall put in a garbage input and see whether the bot repeats the question or is unable to process it and carries on The bot just keeps asking the same question over and over as garbage information is fed to it - until something that makes sense to be processed is passed. Success Sachchit Prasad 02/12/2016
4 Skype Bot SkyScanner - Go back a stage Is the bot able to go back a step should you make an error at a given stage, there are no commands given which say this is the case so I will check if there are any built in commands that let you do this. None of the commnads worked to go backwards, suggesting that a user would have to start over should they make a mistake. Failure Sachchit Prasad 02/12/2016
5 Skype Bot SkyScanner - How dates are handled When prompted for a date to be entered, if the user is unsure and does not select a specific date, are they able to say mroe general statements like a few days, is this handled? The bot is able to handle such queries and saying a "few days" tends to return a date 3 days from the current one and "few weeks" returns a date 3 weeks from the current date. Success Sachchit Prasad 02/12/2016
6 Skype Bot SkyScanner - How dates are handled When prompted for a date to be entered, if the user enters a date too far in the future, does the system search for this or does it recognise that this is note a reasonable date to give - for far forwards can we book? The upper limit for dates is 52 weeks from the current date, anything above this is considered to far away and a relevant error message prompting the user to enter a time closer to the current date is given. Success Sachchit Prasad 02/12/2016
